# Municipal MRIO Korea 2020

This repository contains the Python workflow used to construct a municipality-level multi-regional input-output (MRIO) dataset for Korea in 2020. The workflow covers 229 municipalities and 76 commodity sectors.

## Structure

```text
data/          Construction inputs
metadata/      Region, sector, and account classifications
src/           MRIO construction workflow
diagnostics/   Accounting and benchmark checks
release/       Generated files, ignored by Git
```

All files required by the baseline construction are retained in `data/`, including the alternative distance matrices used in the study.

## Installation

The workflow was tested with Python 3.13 and the package versions listed in `requirements.txt`.

```bash
pip install -r requirements.txt
```

## Construction

```bash
python -m src
```

The command writes three files to `release/`:

```text
release/
├── municipal_mrio_korea_2020.parquet
├── index.csv
└── manifest.json
```

## Diagnostics

Run diagnostics after construction:

```bash
python -m diagnostics
```

The command writes the accounting, benchmark, and production activity checks to:

```text
release/diagnostic_summary.csv
```

## Loading generated output

```python
from src.mrio_io import load_mrio

mrio = load_mrio("release")

mrio.Z
mrio.A
mrio.M
mrio.Y
mrio.Y_d
mrio.Y_m
mrio.VA
mrio.X
```

The loader reads the combined Parquet output generated by the construction
workflow. The technical coefficient matrix `A` is calculated from `Z` and `X`
when accessed. The standalone Zenodo release provides equivalent matrix access
through the included `korea_mrio.py` loader.

## Data release

The finalized dataset and separate matrix files are distributed through Zenodo. This repository provides the construction and diagnostic code required to reproduce and inspect the dataset.
